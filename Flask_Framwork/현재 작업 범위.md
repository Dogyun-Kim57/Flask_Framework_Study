
### 현재까지 작업한 트리 구조
```
minimalapp/
├─ run.py
│  # 프로젝트 실행 진입점
│  # create_app()으로 Flask 앱을 생성한 뒤,
│  # Flask-SocketIO 서버를 실행한다.
│  # 일반 HTTP 요청과 실시간 소켓 통신을 함께 처리하는 시작 파일이다.
│
├─ yolov8n.pt
│  # YOLOv8 경량 모델 가중치 파일
│  # RTSP 카메라 프레임 또는 탐지 입력 프레임에 대해
│  # 객체 탐지를 수행할 때 사용하는 학습 완료 모델 파일이다.
│
├─ app/
│  ├─ __init__.py
│  │  # 애플리케이션 팩토리 파일
│  │  # Flask 앱 생성, Config 적용, 확장 기능 초기화,
│  │  # 블루프린트 등록, SocketIO 이벤트 등록을 담당한다.
│  │  # 현재 구조에서는 다중 카메라 RTSP 제어용 소켓 이벤트도 여기서 연결된다.
│  │
│  ├─ config.py
│  │  # 전체 환경설정 관리 파일
│  │  # SECRET_KEY, DB URI, 메일 설정, 업로드 경로,
│  │  # RTSP 카메라 목록 등 프로젝트 전반 설정값을 중앙 관리한다.
│  │
│  ├─ extensions.py
│  │  # Flask 확장 객체 모음
│  │  # SQLAlchemy, Migrate, Mail, SocketIO 객체를 전역으로 생성해두고
│  │  # 앱 초기화 시 create_app()에서 실제 앱과 연결한다.
│  │
│  ├─ onvif_test.py
│  │  # CCTV/RTSP/ONVIF 확인용 테스트 스크립트
│  │  # 실제 서비스 로직보다는 카메라 스트림 URI 검증, 디버깅, 개발 테스트 목적에 가깝다.
│  │
│  ├─ requirements.txt
│  │  # 프로젝트 실행에 필요한 파이썬 패키지 목록
│  │  # Flask, SQLAlchemy, OpenCV, Torch, Ultralytics, Flask-SocketIO 등
│  │  # 전체 의존성을 관리한다.
│  │
│  ├─ common/
│  │  ├─ __init__.py
│  │  │  # common 패키지 초기화 파일
│  │  │
│  │  ├─ auth.py
│  │  │  # 인증/인가 공통 모듈
│  │  │  # login_required, admin_required 같은 접근 제어 데코레이터와
│  │  │  # 현재 로그인 사용자 정보를 세션에서 꺼내는 헬퍼 함수를 제공한다.
│  │  │
│  │  ├─ exceptions.py
│  │  │  # 커스텀 예외 정의 파일
│  │  │  # 서비스 계층에서 입력 검증 실패나 규칙 위반 시
│  │  │  # ValidationError 같은 예외를 발생시켜 라우트에서 처리할 수 있게 한다.
│  │  │
│  │  ├─ response.py
│  │  │  # 공통 응답 처리 모듈
│  │  │  # render_template, redirect, JSON 응답 형식을 감싸서
│  │  │  # 라우트 코드의 중복을 줄이고 응답 스타일을 통일한다.
│  │  │
│  │  ├─ session_manager.py
│  │  │  # 세션 관리 전담 모듈
│  │  │  # 로그인 성공 시 user_id, username, email, role 등을 세션에 저장하고,
│  │  │  # 로그아웃 시 세션을 정리한다.
│  │  │
│  │  └─ utils.py
│  │     # 공통 유틸리티 함수 모음
│  │     # 문자열 정리, 입력값 전처리 등
│  │     # 여러 서비스에서 반복 사용하는 보조 기능을 제공한다.
│  │
│  ├─ models/
│  │  ├─ __init__.py
│  │  │  # ORM 모델 import 묶음 파일
│  │  │  # Flask-Migrate가 모델을 인식하고,
│  │  │  # 앱 초기화 시 전체 모델을 한 번에 불러오기 쉽게 만든다.
│  │  │
│  │  ├─ user.py
│  │  │  # users 테이블 모델
│  │  │  # 회원 정보, 권한(role), 생성일을 저장한다.
│  │  │  # 문의(contact), 게시글(post) 등 여러 기능의 기준이 되는 핵심 사용자 모델이다.
│  │  │
│  │  ├─ contact.py
│  │  │  # contacts 테이블 모델
│  │  │  # 사용자 문의 내용, 답변(reply), 작성 시각 등을 저장한다.
│  │  │
│  │  ├─ post.py
│  │  │  # posts 테이블 모델
│  │  │  # 게시글 제목, 내용, 작성자, 작성일을 저장한다.
│  │  │  # PostFile과 1:N 관계로 연결된다.
│  │  │
│  │  ├─ post_file.py
│  │  │  # post_files 테이블 모델
│  │  │  # 게시글 첨부파일의 원본 파일명, 저장 파일명, 저장 경로, 파일 크기 등을 저장한다.
│  │  │
│  │  └─ detection.py
│  │     # detections 테이블 모델
│  │     # 탐지된 객체 라벨, confidence, target 일치 여부, alert 여부,
│  │     # camera_id, camera_name, bbox 좌표, detected_at 등을 저장한다.
│  │     # 실시간 탐지 기능을 '조회 가능한 데이터'로 승격시키는 핵심 모델이다.
│  │
│  ├─ repositories/
│  │  ├─ __init__.py
│  │  │  # repositories 패키지 초기화 파일
│  │  │
│  │  ├─ user_repository.py
│  │  │  # User 모델 DB 접근 계층
│  │  │  # 회원 저장, 조회, 관리자 승격, 삭제 같은 DB 작업을 전담한다.
│  │  │
│  │  ├─ contact_repository.py
│  │  │  # Contact 관련 DB 접근 계층
│  │  │  # 문의 저장, 전체 조회, 사용자별 문의 조회 등을 담당한다.
│  │  │
│  │  ├─ post_repository.py
│  │  │  # Post 관련 DB 접근 계층
│  │  │  # 게시글 저장, 검색, 정렬, 페이징, 상세 조회, 수정, 삭제 등을 담당한다.
│  │  │
│  │  ├─ post_file_repository.py
│  │  │  # PostFile 관련 DB 접근 계층
│  │  │  # 첨부파일 저장, 조회, 삭제를 담당한다.
│  │  │
│  │  └─ detection_repository.py
│  │     # Detection 관련 DB 접근 계층
│  │     # 탐지 결과 저장, 전체 탐지 목록 조회, alert 조회,
│  │     # 카메라별 조회, 라벨별 조회, 상세 조회를 담당한다.
│  │     # 즉, 탐지 데이터를 DB와 직접 주고받는 가장 안쪽 계층이다.
│  │
│  ├─ routes/
│  │  ├─ __init__.py
│  │  │  # routes 패키지 초기화 파일
│  │  │
│  │  ├─ main_routes.py
│  │  │  # 메인 홈 화면 라우트
│  │  │  # 첫 진입 페이지를 렌더링하고,
│  │  │  # 로그인 상태에 따라 템플릿에 사용자명 표시 여부를 전달한다.
│  │  │
│  │  ├─ auth_routes.py
│  │  │  # 회원가입 / 로그인 / 로그아웃 라우트
│  │  │  # AuthService와 SessionManager를 사용해 인증 흐름을 처리한다.
│  │  │
│  │  ├─ contact_routes.py
│  │  │  # 문의 등록 및 문의 내역 조회 라우트
│  │  │  # ContactService로 문의를 저장하고,
│  │  │  # 필요 시 MailService로 관리자 메일 발송까지 연결한다.
│  │  │
│  │  ├─ admin_routes.py
│  │  │  # 관리자 전용 일반 관리 라우트
│  │  │  # 문의 목록/상세, 답변 저장, 회원 목록/상세, 관리자 권한 부여,
│  │  │  # 회원 삭제 같은 관리자 기능을 담당한다.
│  │  │
│  │  ├─ member_routes.py
│  │  │  # 일반 회원 전용 라우트
│  │  │  # 내 계정 정보 조회/수정, 내 문의 내역 조회 기능을 처리한다.
│  │  │
│  │  ├─ post_routes.py
│  │  │  # 게시판 관련 라우트
│  │  │  # 게시글 목록, 검색, 상세, 작성, 수정, 삭제,
│  │  │  # 첨부파일 다운로드/삭제, 관리자 게시글 관리까지 담당한다.
│  │  │
│  │  ├─ ai_detect_routes.py
│  │  │  # 실시간 탐지 화면 라우트
│  │  │  # RTSP/IP 카메라 및 웹캠 테스트 화면 진입 템플릿을 렌더링한다.
│  │  │  # 현재는 카메라 목록(config 기반)도 템플릿으로 전달하는 역할을 한다.
│  │  │
│  │  ├─ detection_routes.py
│  │  │  # 탐지 기록 조회 라우트
│  │  │  # DetectionService를 통해 전체 탐지 목록, 카메라별/라벨별 조회,
│  │  │  # 탐지 상세 페이지, alert 목록 페이지를 처리한다.
│  │  │
│  │  └─ detection_admin_routes.py
│  │     # 관리자 전용 탐지 현황 대시보드 라우트
│  │     # DetectionDashboardService가 계산한 통계 데이터를 받아
│  │     # 관리자용 대시보드 템플릿에 전달한다.
│  │
│  ├─ services/
│  │  ├─ __init__.py
│  │  │  # services 패키지 초기화 파일
│  │  │
│  │  ├─ auth_service.py
│  │  │  # 회원가입/로그인 비즈니스 로직
│  │  │  # 입력값 정리, 중복 검사, 비밀번호 해시화, 비밀번호 검증 등을 담당한다.
│  │  │
│  │  ├─ contact_service.py
│  │  │  # 문의 등록 비즈니스 로직
│  │  │  # 이름/이메일/문의내용 검증 후 DB 저장으로 연결한다.
│  │  │
│  │  ├─ mail_service.py
│  │  │  # 관리자 메일 발송 서비스
│  │  │  # 문의 등록 시 관리자 이메일로 문의 내용을 전송한다.
│  │  │
│  │  ├─ post_service.py
│  │  │  # 게시글/첨부파일 비즈니스 로직
│  │  │  # 게시글 생성/수정/삭제, 파일 확장자 검증, 업로드 저장,
│  │  │  # 첨부파일 삭제 등을 담당한다.
│  │  │
│  │  ├─ detection_service.py
│  │  │  # 탐지 결과 비즈니스 로직
│  │  │  # YOLO boxes 결과를 해석하고,
│  │  │  # confidence 필터링, 저장 쿨타임 적용, target 일치 여부 계산,
│  │  │  # alert 여부 판단 후 DetectionRepository로 저장을 위임한다.
│  │  │
│  │  ├─ detection_dashboard_service.py
│  │  │  # 관리자 대시보드용 통계 서비스
│  │  │  # 전체 탐지 수, alert 수, 카메라 수, 라벨 종류 수,
│  │  │  # 카메라별 통계, 라벨별 통계, 최근 탐지 목록을 계산한다.
│  │  │
│  │  └─ ai_stream_service.py
│  │     # 실시간 RTSP 스트림 + YOLO 탐지 핵심 서비스
│  │     # 다중 IP 카메라를 camera_id 기준으로 관리하고,
│  │     # 각 카메라별로 실행 상태, 종료 요청, target 라벨, status를 분리 관리한다.
│  │     # 프레임 수신 → YOLO 추론 → DetectionService 저장 → Socket 프레임 전송
│  │     # → target 일치 시 alert 송신까지 이어지는 중심 서비스다.
│  │
│  ├─ static/
│  │  ├─ css/
│  │  │  ├─ base.css
│  │  │  │  # 전체 기본 스타일 초기화
│  │  │  │
│  │  │  ├─ layout.css
│  │  │  │  # 공통 레이아웃 스타일
│  │  │  │  # header, container, page-top 등 전체 골격을 담당한다.
│  │  │  │
│  │  │  ├─ components.css
│  │  │  │  # 공통 UI 컴포넌트 스타일
│  │  │  │  # 버튼, 입력폼, 드롭다운, 에러 박스 등을 재사용 가능하게 만든다.
│  │  │  │
│  │  │  ├─ home.css
│  │  │  │  # 홈 화면 전용 스타일
│  │  │  │
│  │  │  ├─ auth.css
│  │  │  │  # 로그인/회원가입 전용 스타일
│  │  │  │
│  │  │  ├─ contact.css
│  │  │  │  # 문의 페이지 전용 스타일
│  │  │  │
│  │  │  ├─ admin.css
│  │  │  │  # 관리자 일반 페이지 전용 스타일
│  │  │  │  # 문의 관리, 회원 관리, 목록/상세 카드 UI를 담당한다.
│  │  │  │
│  │  │  ├─ member.css
│  │  │  │  # 회원 전용 페이지 스타일
│  │  │  │
│  │  │  ├─ post.css
│  │  │  │  # 게시판 및 게시글 관련 스타일
│  │  │  │
│  │  │  ├─ ai_stream.css
│  │  │  │  # 실시간 탐지 화면 전용 스타일
│  │  │  │  # 카메라 카드, 상태 표시, 스트림 placeholder, 영상 영역 등을 담당한다.
│  │  │  │
│  │  │  └─ detection_dashboard.css
│  │  │     # 관리자 전용 탐지 현황 대시보드 스타일
│  │  │     # 요약 카드, 통계 패널, 최근 탐지 목록 UI를 담당한다.
│  │  │
│  │  ├─ images/
│  │  │  ├─ main.png
│  │  │  │  # 홈 화면 메인 이미지
│  │  │  │
│  │  │  └─ contact.png
│  │  │     # 문의 관련 화면에 사용하는 시각 요소 이미지
│  │  │
│  │  ├─ js/
│  │  │  └─ ai_detect/
│  │  │     ├─ socket.js
│  │  │     │  # Socket.IO 연결 객체 생성 파일
│  │  │     │  # 다른 JS 모듈들이 공통으로 사용하는 socket 인스턴스를 제공한다.
│  │  │     │
│  │  │     ├─ main.js
│  │  │     │  # 실시간 탐지 페이지 JS 초기화 파일
│  │  │     │  # 아래 세부 모듈들을 한 번에 초기화하는 진입점 역할을 한다.
│  │  │     │
```
