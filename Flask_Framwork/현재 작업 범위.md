
### 현재까지 작업한 트리 구조
```
minimalapp/
├─ run.py
│  # 프로젝트 실행 진입점
│  # create_app()으로 Flask 앱을 생성한 뒤,
│  # SocketIO 서버를 실행하여 일반 웹 요청 + 실시간 소켓 통신을 함께 처리한다.
│
├─ yolov8n.pt
│  # YOLOv8 경량 모델 가중치 파일
│  # 실시간 CCTV 스트림 프레임에 대해 객체 탐지를 수행할 때 사용한다.
│
├─ app/
│  ├─ __init__.py
│  │  # 애플리케이션 팩토리 파일
│  │  # Flask 앱 생성, Config 적용, 확장 기능(db/migrate/mail/socketio) 초기화,
│  │  # 블루프린트 등록, Socket 이벤트 등록까지 담당한다.
│  │
│  ├─ config.py
│  │  # 전체 환경설정 관리 파일
│  │  # SECRET_KEY, DB 연결 정보, 메일 설정, 업로드 경로,
│  │  # RTSP_URL 같은 CCTV/실시간 탐지 관련 설정값도 여기서 관리한다.
│  │
│  ├─ extensions.py
│  │  # Flask 확장 객체를 한곳에 모아두는 파일
│  │  # SQLAlchemy, Migrate, Mail, SocketIO 객체를 전역으로 생성하여
│  │  # 앱 초기화 시 재사용할 수 있게 한다.
│  │
│  ├─ onvif_test.py
│  │  # CCTV 장비의 ONVIF 정보를 통해 실제 스트림 URI를 확인하기 위한 테스트용 파일
│  │  # 개발/디버깅용 보조 스크립트이며, 실서비스 로직과는 분리된 확인 도구이다.
│  │
│  ├─ requirements.txt
│  │  # 현재 app 폴더 기준 의존성 기록 파일
│  │  # Flask, OpenCV, Torch, Ultralytics, SocketIO 등 실행에 필요한 패키지를 관리한다.
│  │
│  ├─ common/
│  │  ├─ __init__.py
│  │  │  # common 패키지 초기화 파일
│  │  │
│  │  ├─ auth.py
│  │  │  # 인증/인가 공통 처리 모듈
│  │  │  # login_required, admin_required 데코레이터와
│  │  │  # 현재 로그인 사용자 정보 조회 함수들을 제공한다.
│  │  │
│  │  ├─ exceptions.py
│  │  │  # 커스텀 예외 정의 모듈
│  │  │  # 입력값 검증 실패나 비즈니스 규칙 위반 시 ValidationError 등을 발생시킨다.
│  │  │
│  │  ├─ response.py
│  │  │  # 공통 응답 처리 모듈
│  │  │  # render_template, redirect, JSON 응답 형식을 통일해서
│  │  │  # 라우트 코드의 중복을 줄여준다.
│  │  │
│  │  ├─ session_manager.py
│  │  │  # 세션 저장/조회 전담 모듈
│  │  │  # 로그인 성공 시 사용자 id, username, email, role 등을 세션에 저장하고,
│  │  │  # 로그아웃 시 세션을 정리한다.
│  │  │
│  │  └─ utils.py
│  │     # 공통 유틸리티 모음
│  │     # 문자열 정리(clean_text) 등 서비스 계층에서 반복 사용하는 보조 기능을 제공한다.
│  │
│  ├─ models/
│  │  ├─ __init__.py
│  │  │  # ORM 모델 import 묶음 파일
│  │  │  # Flask-Migrate나 앱 초기화 시 모델 인식을 쉽게 해준다.
│  │  │
│  │  ├─ user.py
│  │  │  # users 테이블 모델
│  │  │  # 회원 정보(아이디, 이메일, 비밀번호 해시, 권한, 생성일)를 저장하며
│  │  │  # 문의/게시글과의 관계의 중심이 되는 핵심 모델이다.
│  │  │
│  │  ├─ contact.py
│  │  │  # contacts 테이블 모델
│  │  │  # 사용자 문의 정보, 답변 내용, 작성 시각 등을 저장한다.
│  │  │
│  │  ├─ post.py
│  │  │  # posts 테이블 모델
│  │  │  # 게시글 제목, 내용, 작성자, 생성일을 저장하고
│  │  │  # 첨부파일(PostFile)과 1:N 관계를 가진다.
│  │  │
│  │  └─ post_file.py
│  │     # post_files 테이블 모델
│  │     # 게시글 첨부파일의 원본 파일명, 저장 파일명, 경로, 크기 등을 저장한다.
│  │
│  ├─ repositories/
│  │  ├─ __init__.py
│  │  │  # repositories 패키지 초기화 파일
│  │  │
│  │  ├─ user_repository.py
│  │  │  # User 모델에 대한 DB 접근 전용 계층
│  │  │  # 회원 저장, 조회, 관리자 승격, 삭제 등의 쿼리를 담당한다.
│  │  │
│  │  ├─ contact_repository.py
│  │  │  # Contact 모델에 대한 DB 접근 전용 계층
│  │  │  # 문의 등록, 목록 조회, 사용자별 조회 등 문의 관련 쿼리를 담당한다.
│  │  │
│  │  ├─ post_repository.py
│  │  │  # Post 모델에 대한 DB 접근 계층
│  │  │  # 게시글 저장, 검색, 정렬, 페이징, 상세 조회, 수정, 삭제를 담당한다.
│  │  │
│  │  └─ post_file_repository.py
│  │     # PostFile 모델에 대한 DB 접근 계층
│  │     # 첨부파일 저장, 조회, 삭제를 담당한다.
│  │
│  ├─ routes/
│  │  ├─ __init__.py
│  │  │  # routes 패키지 초기화 파일
│  │  │
│  │  ├─ main_routes.py
│  │  │  # 메인 홈 화면 라우트
│  │  │  # 로그인 상태에 따라 사용자명 표시 여부를 결정하고
│  │  │  # 첫 진입 페이지를 렌더링한다.
│  │  │
│  │  ├─ auth_routes.py
│  │  │  # 로그인 / 회원가입 / 로그아웃 라우트
│  │  │  # AuthService와 SessionManager를 사용해 인증 흐름을 처리한다.
│  │  │
│  │  ├─ contact_routes.py
│  │  │  # 문의 등록 및 문의 내역 조회 라우트
│  │  │  # ContactService로 문의를 저장하고,
│  │  │  # MailService를 통해 관리자 메일 발송까지 연결한다.
│  │  │
│  │  ├─ admin_routes.py
│  │  │  # 관리자 전용 라우트
│  │  │  # 전체 문의 목록, 문의 답변 저장, 회원 목록/상세 조회,
│  │  │  # 관리자 권한 부여, 회원 삭제 기능 등을 담당한다.
│  │  │
│  │  ├─ member_routes.py
│  │  │  # 일반 회원 전용 라우트
│  │  │  # 내 계정 정보 조회/수정, 내 문의 내역 조회 기능을 처리한다.
│  │  │
│  │  ├─ post_routes.py
│  │  │  # 게시판 관련 라우트
│  │  │  # 게시글 목록, 검색, 상세, 작성, 수정, 삭제,
│  │  │  # 첨부파일 다운로드/삭제, 관리자 게시글 관리 기능을 담당한다.
│  │  │
│  │  └─ ai_detect_routes.py
│  │     # 실시간 탐지 페이지 라우트
│  │     # CCTV 스트림을 보여주는 ai_stream.html 페이지를 렌더링하며,
│  │     # 실시간 탐지 기능의 웹 진입점 역할을 한다.
│  │
│  ├─ services/
│  │  ├─ __init__.py
│  │  │  # services 패키지 초기화 파일
│  │  │
│  │  ├─ auth_service.py
│  │  │  # 회원가입/로그인 비즈니스 로직 처리 모듈
│  │  │  # 입력값 정리, 중복 검사, 비밀번호 해시화, 비밀번호 검증 등을 담당한다.
│  │  │
│  │  ├─ contact_service.py
│  │  │  # 문의 등록 비즈니스 로직 모듈
│  │  │  # 이름/이메일/문의 내용 검증 후 DB 저장으로 연결한다.
│  │  │
│  │  ├─ mail_service.py
│  │  │  # 관리자 메일 발송 서비스
│  │  │  # 문의가 접수되면 관리자 이메일로 문의 내용을 전송한다.
│  │  │
│  │  ├─ post_service.py
│  │  │  # 게시글/첨부파일 비즈니스 로직 모듈
│  │  │  # 게시글 생성/수정/삭제, 파일 확장자 검증, 업로드 저장,
│  │  │  # 첨부파일 삭제 등을 담당한다.
│  │  │
│  │  └─ ai_stream_service.py
│  │     # 실시간 CCTV 스트림 + YOLO 객체 탐지 핵심 서비스
│  │     # RTSP URL로 CCTV 스트림에 접속하고,
│  │     # 프레임을 읽어 YOLOv8으로 탐지 후,
│  │     # SocketIO로 프론트엔드에 영상 프레임과 탐지 알림을 실시간 전송한다.
│  │
│  ├─ static/
│  │  ├─ css/
│  │  │  ├─ base.css
│  │  │  │  # 전체 기본 스타일 초기화
│  │  │  │
│  │  │  ├─ layout.css
│  │  │  │  # 공통 레이아웃 스타일
│  │  │  │  # header, container, page-top 등 전체 틀을 담당한다.
│  │  │  │
│  │  │  ├─ components.css
│  │  │  │  # 공통 컴포넌트 스타일
│  │  │  │  # 버튼, 폼, 드롭다운, 에러 메시지 등의 재사용 UI를 담당한다.
│  │  │  │
│  │  │  ├─ home.css
│  │  │  │  # 홈 화면 전용 스타일
│  │  │  │
│  │  │  ├─ auth.css
│  │  │  │  # 로그인/회원가입 화면 전용 스타일
│  │  │  │
│  │  │  ├─ contact.css
│  │  │  │  # 문의하기/문의완료 화면 전용 스타일
│  │  │  │
│  │  │  ├─ admin.css
│  │  │  │  # 관리자 페이지 전용 스타일
│  │  │  │
│  │  │  ├─ member.css
│  │  │  │  # 회원 전용 페이지 스타일
│  │  │  │
│  │  │  ├─ post.css
│  │  │  │  # 게시판/첨부파일/목록/상세 화면 스타일
│  │  │  │
│  │  │  └─ ai_stream.css
│  │  │     # 실시간 탐지 페이지 전용 스타일
│  │  │     # CCTV 영상 출력 영역, 탐지 타겟 입력창, 알림 박스 UI를 담당한다.
│  │  │
│  │  ├─ js/
│  │  │  └─ ai_stream.js
│  │  │     # 실시간 탐지 페이지 전용 JavaScript
│  │  │     # SocketIO 연결, 실시간 프레임 수신, 타겟 전송,

│  │  │
│  │  ├─ images/

│  │  │
│  │  └─ uploads/
│  │     # 게시판 첨부파일 업로드 저장 폴더
```
